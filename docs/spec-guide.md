[기술 명세서 작성 원칙]

1. 구현 코드가 아닌 스펙 정의에 집중한다
2. 정보는 테이블 형식으로 정리한다 (파라미터, 반환 타입, 필드, 에러 코드 등)
3. 코드 블록은 최소화한다 (시그니처, 데이터 구조, Enum 정의 정도만 허용)
4. 개발자가 이 문서만 보고 구현할 수 있어야 한다

[제외 항목]
- 구현 로직, 알고리즘, 동작하는 코드 블록

[문서 구성]
필수: 개요, 데이터/모델 설계, 예외 처리
선택: 데이터 소스 분석, 패키지 구조, 구현 우선순위, 참고 자료

[수정 시 원칙]
1. 변경/추가된 부분만 작성한다 (전체 재작성 금지)
2. 문서 상단에 변경 이력 섹션을 추가/갱신한다
3. 본문에서 변경된 부분은 해당 위치에 직접 반영한다
